# Chaos Board

## 개요
찬반 투표와 실시간 토론을 하나의 화면에서 해결하는 웹 앱.

## 기술 스택
- Next.js (App Router)
- React 18, TypeScript
- Tailwind CSS

## 주요 명령어
- `npm run dev` - 개발 서버 실행 (port 3000)
- `npm run build` - 프로덕션 빌드
- `npm start` - 프로덕션 서버 실행

## 코드 규칙

### 필수 규칙 (반드시 따르기)
- **No else**: guard clause / early return 사용. `else` 또는 `else if` 절대 사용 금지.
- **Const-first**: 기본 `const` 사용. 재할당이 꼭 필요할 때만 `let` 사용. `var` 금지.
- **No magic values**: 의미 있는 숫자/문자열은 상수로 추출.
- **Clear naming**: 이름은 역할과 책임이 드러나야 함. `util`, `data`, `handler`, `temp` 같은 모호한 이름 금지.

### 작업 규칙
- 항상 최소 변경으로 문제 해결.
- 관련 없는 파일 수정 금지.
- 기존 구조와 진입점 임의로 바꾸지 않기.
- 새 라이브러리는 반드시 필요할 때만 도입.
- 기존 패턴 먼저 따르기. 반복되는 문제나 명확한 구조적 문제가 있을 때만 새 패턴 제안.
- 리팩토링과 동작 변경은 가능하면 분리.

### 계획 필요
아래 작업은 구현 전에 계획 필수:
- 기능 추가
- 동작 변경
- 다중 파일 수정
- 아키텍처 또는 데이터 흐름 변경
- API/인증/스토리지/타이머/폴링 변경
- 테스트 전략 변경
- 요구사항이 모호한 작업
- 영향 범위가 한눈에 보이지 않는 리팩토링

### 계획 불필요
- 오타 수정
- 문서 문구 수정
- 동작 변화가 없는 국소적 리팩토링
- 명백한 단일 파일 버그 수정
